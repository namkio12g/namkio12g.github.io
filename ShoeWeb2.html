Shoe website v 2.0

This web application is a comprehensive solution for storing and managing products, orders, and other data for a shoe store, built using an MVC (Model-View-Controller) architecture. It serves as a substantial upgrade to both the backend and frontend of the original website, enhancing its functionality, performance, and user experience

Key components

  • Authentication using Jwt: I implemented JWT to generate tokens containing staff login information, with an expiration time set for each token. This approach ensures that even if a staff member closes the tab, the token remains valid for a specified period, allowing them to resume their session without needing to log in again within that timeframe.
  • Authorization: I initially planned to use AuthorizeApi for role-based permission checks on each API route. However, I opted for view filtering instead to hide functionalities entirely from unauthorized roles, enhancing security by concealing restricted features.
  • Data Management(for staff): With the appropriate permission ,User can create ,delete ,update and query the data from database
  • Customer experience: In my app, customers can easily view and filter products by type, brand, and size; add items to their cart to create orders; and check details as well as track the status of their past and current orders.

Techonologies used

  • Front-End: Boostrap, HTML, CSS, Pug, Javascript.
  • Back-End: Nodejs (framework express), Restful Api
  • Database: MongoDB
